# Развёртывание Jewerly of Soul на reg.ru

Пошаговая инструкция: как выложить сайт с настоящим backend на хостинг.
Технических знаний не требует — всё делается через панель управления reg.ru.

---

## Что нужно

- Купленный домен (например `jewerly-of-soul.ru`).
- Купленный тариф «Виртуальный хостинг» с поддержкой **PHP** и **MySQL**.
- Папка проекта `Jewerly_of_Soul` со всеми файлами — она у тебя уже есть.

---

## Шаг 1. Загрузить файлы сайта на хостинг

1. Зайди в панель управления reg.ru → раздел своего хостинга → **Файловый менеджер**
   (или подключись по FTP — данные FTP есть в письме от reg.ru).
2. Открой корневую папку сайта. Обычно она называется `public_html`, `www` или
   совпадает с именем домена.
3. Загрузи в неё **всё содержимое** папки `Jewerly_of_Soul`: файлы `*.html`,
   папки `api/`, `js/`, `styles/`, `assets/`, `data/`, файл `sw.js`,
   `manifest.webmanifest` и остальное.

Должно получиться так: `index.html` лежит в корне сайта, рядом с ним — папка `api/`.

---

## Шаг 2. Создать базу данных MySQL

1. В панели reg.ru найди раздел **«Базы данных MySQL»** (или «MySQL»).
2. Нажми **создать базу данных**. Задай пароль.
3. **Запиши три значения** — они понадобятся на следующем шаге:
   - имя базы данных (например `u1234567_jos`);
   - имя пользователя базы (часто совпадает с именем базы);
   - пароль базы.

---

## Шаг 3. Создать таблицы

1. В панели reg.ru открой **phpMyAdmin** (кнопка рядом с базой данных).
2. Слева выбери свою базу данных.
3. Вверху открой вкладку **SQL**.
4. Открой файл `api/schema.sql` (любым текстовым редактором), скопируй весь его
   текст, вставь в поле и нажми **Вперёд** / **Выполнить**.
5. Должны появиться четыре таблицы: `users`, `sessions`, `orders`, `ideas`.
6. *(Необязательно)* Чтобы лента сообщества не была пустой на старте,
   тем же способом выполни файл `api/seed-community.sql` — он добавит
   10 демо-авторов и 30 готовых идей. Хочешь начать с чистой ленты —
   просто пропусти этот шаг.

---

## Шаг 4. Прописать доступы к базе

1. В файловом менеджере открой файл `api/config.php` для редактирования.
2. Впиши значения из Шага 2:
   ```
   'db_name' => 'имя_твоей_базы',
   'db_user' => 'имя_пользователя_базы',
   'db_pass' => 'пароль_базы',
   ```
   Поле `db_host` оставь `localhost`.
3. В поле `admin_email` впиши свою почту — на неё ты потом зарегистрируешься,
   и она автоматически получит доступ к админ-панели.
4. Сохрани файл.

---

## Шаг 5. Включить SSL (адрес https://)

В панели reg.ru найди раздел **SSL** и подключи **бесплатный сертификат
Let's Encrypt** для своего домена. Он продлевается автоматически и навсегда
бесплатно. После подключения сайт будет открываться по `https://`.

---

## Шаг 6. Проверить, что backend работает

Открой в браузере адрес:

```
https://твой-домен.ru/api/?route=ping
```

- Если видишь `{"ok":true,"time":"..."}` — backend работает, всё настроено верно.
- Если видишь ошибку про базу данных — вернись к Шагу 4 и перепроверь
  значения в `api/config.php`.

---

## Шаг 7. Стать администратором

1. Открой сайт, перейди в **Регистрацию**.
2. Зарегистрируйся, указав **ту же почту**, что вписана в `admin_email`
   (Шаг 4).
3. Теперь тебе доступна панель заявок: `https://твой-домен.ru/admin.html`

Там будут появляться все заявки из конструктора — с возможностью менять
статус и оставлять заметки.

---

## Как это устроено

- На «боевом» домене сайт автоматически работает через backend: настоящие
  аккаунты, заявки в базе данных.
- На `localhost` и GitHub Pages (где backend нет) сайт сам переключается
  в режим предпросмотра на памяти браузера — менять ничего не нужно.
- Пароли в базе хранятся в виде необратимого хеша.
- Служебные файлы (`schema.sql`, эта инструкция) закрыты от скачивания
  через `api/.htaccess`.

---

## Если что-то не так

- **Страницы открываются, но вход/регистрация выдают ошибку связи** —
  проверь Шаг 6 (`?route=ping`). Скорее всего дело в `api/config.php`.
- **`?route=ping` выдаёт ошибку базы** — неверные имя базы / пользователь /
  пароль в `config.php`, либо не выполнен Шаг 3 (таблицы не созданы).
- **Тариф не поддерживает MySQL** — обратись в поддержку reg.ru и попроси
  тариф виртуального хостинга с PHP и MySQL (есть даже на младших тарифах).
- **admin.html пишет «доступ только у администратора»** — ты зарегистрировался
  не на ту почту, что в `admin_email`. Впиши в `config.php` свою почту и
  зарегистрируйся заново (или поменяй её у существующего пользователя).
